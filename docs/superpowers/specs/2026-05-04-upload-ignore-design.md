# 微信小程序上传包忽略项设计

**目标：** 在不影响运行功能的前提下，避免将测试和设计文档目录打进微信小程序上传包。

## 范围

- 修改 `project.config.json`
- 仅配置上传忽略目录：
  - `tests/`
  - `docs/`
- 不修改业务代码
- 不调整页面、样式、权限或运行时逻辑

## 方案

采用微信开发者工具项目配置中的 `packOptions.ignore`：

- 在上传时忽略 `tests/` 目录
- 在上传时忽略 `docs/` 目录
- 保留现有 `include` 为空，不改变其它打包策略

## 预期结果

- 本地开发文件仍保留在项目中
- 上传体验证时，这两个目录不进入小程序包
- 运行代码目录 `pages/`、`utils/`、应用配置文件不受影响
